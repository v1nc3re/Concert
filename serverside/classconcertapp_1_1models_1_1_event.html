Detailed Description

create_concert_user is bound to the post_save signal.

So everytime a model object gets saved, the create_concert_user checks if it's a User that is getting saved, then creates the ConcertUser An abstract class (abstract by Concert semantics, not Django) used to house information for all the events that can occur on the system. Built for logging purpose, not event triggered events, although that functionality could theoretically work using django signals.
